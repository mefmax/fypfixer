# Project Reports

This directory contains all project reports for stakeholders.

## Report Types

### Product Manager Reports
Prefix: `REPORT_PM_`

Content:
- Executive summaries
- Feature completion status
- User flows and journeys
- Metrics and KPIs
- Next steps and recommendations
- Deployment readiness

Audience: Product Managers, Business stakeholders

### Technical Architecture Reports
Prefix: `REPORT_ARCHITECT_`

Content:
- System architecture
- Technology stack details
- Database schemas
- API specifications
- Security considerations
- Scalability strategy
- Technical debt tracking

Audience: Technical Architects, Engineering Leads

## Naming Convention

```
REPORT_<TYPE>_<Date/Session>.txt

Examples:
- REPORT_PM_Day3_Evening.txt
- REPORT_ARCHITECT_Day3_Evening.txt
- REPORT_PM_Sprint1_Final.txt
```

## Guidelines

1. **Format**: Plain text (.txt) or Markdown (.md)
2. **Frequency**: After each major milestone or sprint
3. **Storage**: Keep all historical reports for reference
4. **Distribution**: Share with respective stakeholders via email or Slack

## Report Generation

Reports are generated by the development team (with Claude Code assistance)
after significant development sessions or milestones.

To request a report:
- Ask the developer to generate a PM report for product updates
- Ask the developer to generate an Architect report for technical details
